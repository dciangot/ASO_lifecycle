PhEDEx Javascript examples

See https://twiki.cern.ch/twiki/bin/view/CMS/PhedexProjWebsite for the accompanying documentation. These examples are coded with the assumption that you have the YUI toolkit installed locally, and are running the PhEDEx proxy-server configured to use it. See https://twiki.cern.ch/twiki/bin/view/CMS/PhEDExWebsiteDeveloperGuide for the documentation about that.

Source code for these examples is in CVS in PhEDExWeb/ApplicationServer/examples

These are the original, YUI-2 based examples

  • Use the Loader to load a YUI class and do something when it arrives
  • Use the Logger with access to the log-to-server controls
  • Use the Profiler with a few sample objects churning up CPU
  • Use the History manager to navigate backwards and forwards while maintaining internal application state
  • Load a complete but empty application i.e. with the core, sandbox, logger, profiler, everything except the parts that rely on PhEDEx data itself
  • Load the "Nodes" module with the full core/sandbox environment to create it
  • Load the "Agents" module with the full core/sandbox environment to create it
  • Load the "LinkView" module with the full core/sandbox environment to create it
  • Load the "Agents" and "Nodes" modules on the same page, showing how to use the core to fine-tune their configuration and strip out unwanted features

  • Call the dataservice at the lowest-level, setting up the call directly with events for callbacks
  • Call the dataservice passing a callback function
  • Call the dataservice and handle the error caused by a non-existing API
  • Call the dataservice using the standard core to drive it all with notifications to a fake module
  • Call the dataservice using the standard core and a real but stripped phedex-module

  • Load the "protovis" standalone demo module
  • Load the "storageusage" module
  • Call the dataservice and use jquery to set up the parameters passing a requestsetstatefake function, this should be the FSM prototype
